# 🌿 EcoMonitor — Застосунок для екологічного моніторингу повітря в Ужгороді

**EcoMonitor** — це автоматизований вебсервіс на базі Java та Spring Boot, призначений для збору, збереження, геопросторової прив'язки та графічного відображення екологічних і метеорологічних параметрів атмосферного повітря в реальному часі для міста Ужгород.

Проєкт розроблено в рамках курсової роботи з дисципліни «Об'єктно-орієнтоване програмування» студентами Ужгородського національного університету (УжНУ).

---

## 🚀 Ключовий функціонал

* **📍 Автоматична геолокація:** Інтеграція з W3C Geolocation API дозволяє застосунку зчитувати координати користувача через браузер і динамічно підвантажувати дані найближчої екологічної станції.
* **🕒 Фоновий планувальник завдань:** Серверний модуль за допомогою `@Scheduled` кожні 10 хвилин асинхронно опитує OpenWeather API для всіх опорних точок міста.
* **📈 Серверна генерація графіків:** Аналітичний модуль на базі бібліотеки JFreeChart транслює двовимірні графіки динаміки показників (останні 20 вимірів) у вигляді бінарного PNG-потоку безпосередньо з оперативної пам'яті сервера.
* **🎨 Динамічний інтерфейс за шкалою AQI:** Картки показників динамічно змінюють свій колір (зелений/жовтий/червоний) на основі міжнародного індексу якості повітря за принципом найгіршого показника (*Worst-Case Scenario*).
* **📋 Хронологічний архів:** Збереження повної історії екологічних зрізів у базі даних PostgreSQL та виведення у зручному адаптивному форматі (CSS Grid).

---

## 🛠️ Технологічний стек

* **Backend:** Java 17, Spring Boot 3.x (Spring MVC, Spring Data JPA)
* **Database & ORM:** PostgreSQL, Hibernate ORM
* **Frontend:** Thymeleaf (Server-Side Rendering), Bootstrap 5.3, JavaScript (W3C Geolocation API), CSS3 Custom Properties
* **Аналітика та API:** JFreeChart, OpenWeather Air Pollution & Weather API, RestTemplate

---

## 🏛️ Опорні екологічні станції в Ужгороді
Система підтримує моніторинг у 4 ключових мікрорайонах міста:
1. **Центр** (48.6212, 22.2978)
2. **БАМ (УжНУ)** (48.6341, 22.2882)
3. **Шахта** (48.6385, 22.3122)
4. **Радванка** (48.6110, 22.3280)

Вибір станції сервером базується на обчисленні мінімальної евклідової відстані від координат користувача.

## 📦 Інструкція із розгортання та запуску

### 1. Передумови
Переконайтеся, що на вашому комп'ютері встановлено:
* **JDK 17** або вище
* **PostgreSQL** (створіть порожню базу даних з назвою `ecomonitor`)
* **Maven** (або використовуйте вбудований у проєкт `mvnw`)

### 2. Конфігурація
Створіть або відредагуйте файл `src/main/resources/application.properties` у вашому проєкті:

```properties
spring.datasource.url=jdbc:postgresql://localhost:5123/ecomonitor
spring.datasource.username=ваш_loгін_postgres
spring.datasource.password=ваш_пароль_postgres
spring.jpa.hibernate.ddl-auto=update

# Конфігурація API OpenWeather
api.openweathermap.key=ВАШ_ПЕРСОНАЛЬНИЙ_API_KEY
api.location.lat=48.6212
api.location.lon=22.2978
